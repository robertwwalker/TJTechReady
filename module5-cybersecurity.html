Cyberattacks cost organizations billions annually — and most breaches start with human error. This module gives you the knowledge to protect yourself and your workplace.

Why This Matters

You are the first line of defense

Organizations spend millions on firewalls and antivirus software, but the most common entry point for attackers is a person clicking the wrong link. According to cybersecurity reports, over 90% of successful cyberattacks start with phishing — a deceptive email or message tricking someone into taking a harmful action.

This isn't about blaming employees. Attackers are sophisticated and their methods are increasingly convincing. The goal is to build habits that make you a harder target and know what to do when something seems wrong.

Analogy
Think of cybersecurity like office building security. The company has locks, security cameras, and a reception desk (technical controls). But if an employee holds the door open for someone without a badge because they seem friendly, all that technical security is bypassed instantly. Human behavior is the security perimeter.

Passwords

Strong passwords & why they matter

Weak passwords are one of the leading causes of account compromises. "123456" is still the most common password in data breaches. A strong password protects your accounts even if someone has your email address.

TEST YOUR PASSWORD STRENGTH:

Enter a password above to see its strength
❌
Never use: Your name, birthday, pet's name, company name, "password", "123456", or any single dictionary word. These are cracked in seconds by automated tools.
⚠️
Avoid: Reusing the same password across multiple sites. If one site is breached, attackers try your password on every other site (credential stuffing).
✅
Best practice: Use a password manager (Bitwarden, 1Password, LastPass). It generates and stores unique, complex passwords for every site. You only remember one master password.
Multi-Factor Authentication

MFA — Your most powerful account protection

Multi-Factor Authentication (MFA) requires two or more forms of verification to log in: something you know (password) + something you have (phone/authenticator app) or something you are (fingerprint). Even if someone steals your password, they still can't log in without your second factor.

Microsoft reports that MFA blocks 99.9% of automated account attacks. It's the single most impactful thing you can do for account security. Always enable MFA on work accounts, email, and any service containing sensitive information.

Authenticator App
SMS Text Code
Physical Key
Biometrics

Most secure and recommended. Apps like Microsoft Authenticator, Google Authenticator, or Authy generate a 6-digit code that changes every 30 seconds. Even if attackers capture the code, it expires in half a minute. No SIM card required — works offline.

Better than nothing, but not ideal. A text message with a code sent to your phone. Vulnerable to SIM-swapping attacks (where criminals convince your phone carrier to transfer your number to their SIM). Also vulnerable to phishing pages that ask for your SMS code in real-time. Still far better than password-only login.

Strongest option for high-security accounts. A physical USB device (like a YubiKey) that you plug in to confirm your identity. Cannot be phished remotely — the attacker would need to physically have the key. Used by journalists, executives, and anyone with very high-security needs.

Convenient and increasingly common. Fingerprint readers, Face ID, and Windows Hello use physical characteristics. Very hard to steal remotely. Used as the second factor on phones and modern laptops. Combine with a PIN (not just biometric alone) for best security.

Recognizing Phishing

Red flags in emails and messages

Phishing attacks have become sophisticated — some are nearly indistinguishable from real emails. But most still have telltale signs.

Email red flags
Link verification
Urgency tactics
What to do
🚩
Sender address mismatch: The display name says "Microsoft Support" but the actual email address is "support@m1crosoft-help.net". Always hover over or click the sender name to see the real address.
🚩
Generic greetings: "Dear Customer" or "Dear User" instead of your actual name suggests a mass phishing campaign that doesn't know who you are.
⚠️
Unexpected attachments: An invoice from a company you don't recognize, or a file you weren't expecting. .exe, .zip, .docm (macro-enabled) files are particularly risky.
⚠️
Poor grammar/spelling: While sophisticated attacks are well-written, many phishing emails still contain errors. Awkward phrasing or odd word choices are warning signs.

Before clicking any link in email: hover over it (don't click) and look at the URL that appears in the bottom of your browser or as a tooltip. Verify: (1) Is the domain exactly correct? (2) Does it use HTTPS? (3) Is the domain relevant to who supposedly sent the email? A PayPal email linking to paypal-secure-login.xyz is not from PayPal.

On mobile, long-press a link to preview the URL before tapping. When in doubt, navigate to the website manually by typing the address in your browser instead of clicking the link.

Phishing emails create artificial urgency to prevent you from thinking clearly: "Your account will be suspended in 24 hours," "URGENT: Verify your payment immediately," "Your file will be deleted unless you confirm now."

Legitimate companies almost never demand immediate action via email that bypasses normal processes. If something seems urgent, go to the company's official website directly (type it yourself) and log in there — don't click the email link.

If you receive a suspicious email: (1) Don't click any links or download attachments. (2) Don't reply to it. (3) Report it to IT using your company's reporting mechanism (often a button in Outlook: "Report Phishing"). (4) If you think it might be real (e.g., it claims to be from your bank), contact the organization directly using a phone number or website address you already know — not contact info in the suspicious email.

If you already clicked: immediately report it to IT. Early detection limits damage. Don't be embarrassed — phishing attacks fool even security professionals.

Data Privacy

Personal & company data — your responsibilities

Data privacy is both a legal obligation (GDPR, HIPAA, CCPA) and an ethical one. As an employee, you handle data about customers, colleagues, and the business. Mishandling it has serious consequences.

What counts as sensitive data
Safe data handling
Regulations overview
Breach response

Personally Identifiable Information (PII): Name, address, phone number, email, date of birth, Social Security/national ID numbers, passport numbers, financial account details. This information, in the wrong hands, enables identity theft.

Protected Health Information (PHI): Medical records, diagnoses, treatment history. Protected by HIPAA in the US.

Company confidential data: Financial results before public announcement, client lists, pricing models, strategic plans, source code, employee salaries. This data can harm the company if leaked.

Key habits: (1) Need to know — only access data you actually need for your job. Don't explore customer records out of curiosity. (2) Encrypt sensitive files before emailing — check if your email system auto-encrypts or use the built-in encryption options. (3) Lock your screen (Win+L or Ctrl+Cmd+Q on Mac) when stepping away from your desk. (4) Don't store sensitive data in personal cloud accounts — use company-approved tools only. (5) Use company devices for company data — avoid emailing work files to personal accounts.

GDPR (EU): Requires explicit consent to collect personal data, the right to be forgotten, data breach notification within 72 hours. Applies to any organization handling EU residents' data.

HIPAA (US): Protects health information. Healthcare workers and their business associates must safeguard PHI. Violations carry heavy fines.

CCPA (California): Gives California residents rights over their personal data — to know what's collected, to opt out of sales, to request deletion.

You don't need to be a lawyer — but know your company's policies and when to escalate to the legal/compliance team with data-related questions.

If you think a data breach has occurred (you accidentally sent sensitive data to the wrong person, you can't find a laptop, you suspect your account is compromised): Report it immediately. Don't wait and hope it resolves itself. Most regulations require organizations to report breaches within 72 hours — the clock starts when the breach is discovered, so early reporting gives the company maximum time to respond. You won't be penalized for reporting — you may be penalized for not reporting.

💻 Interacting with code — a foundation for AI

You don't need to be a programmer to work with AI tools effectively. But understanding a few code concepts removes major barriers — especially when running AI scripts, reading error messages, or using the command line.

Reading code
The terminal
Error messages
Config files

You don't need to write code to read it. When you encounter an AI script or configuration, here's what to look for:

# Lines starting with # are comments — they explain what the code does
# They don't run — they're notes for humans
model_name = "claude-sonnet-4-5" # This sets which AI model to use
max_tokens = 1000 # Maximum length of the AI response
input_file = "data.csv" # The file to process
print("Processing started...") # Shows a message when run

Key patterns to recognize: variable = value (setting a configuration), function(input) (doing something with data), import library (loading a tool), and # comments explaining what things do. These patterns are consistent across Python, JavaScript, and most AI scripting languages.

The terminal (Command Prompt on Windows, Terminal on Mac/Linux) is a text interface for your computer. Many AI tools are run from here. You don't need to memorize it — just know enough to not panic.

cd Documents — Change Directory. Move into a folder. Like double-clicking a folder in File Explorer, but in text.
ls (Mac/Linux) or dir (Windows) — List files in the current folder. Tells you where you are.
python script.py — Run a Python script. Replace script.py with the actual filename. Most AI tools are run this way.
pip install package-name — Install a Python library. Like downloading an app, but for code components AI tools depend on.
Ctrl+C — Stop a running program. If something is running and you need to cancel, press Ctrl+C in the terminal.

The terminal always has a current working directory — the folder it's "inside." File paths you type are relative to this folder. Use pwd (Mac/Linux) or cd (Windows) to see where you are.

Error messages look scary but almost always tell you exactly what's wrong. Learn to read them from the bottom up — the last few lines are usually the actual error; the top lines are just the chain of calls that led there.

ModuleNotFoundError: No module named 'anthropic'
Meaning: A required library isn't installed. Fix: Run pip install anthropic in the terminal.
FileNotFoundError: [Errno 2] No such file or directory: 'data.csv'
Meaning: The script can't find the file at the specified path. Fix: Check the file exists and the path is correct. The terminal's current directory might not be where you think.
AuthenticationError: 401 Invalid API key
Meaning: Your API key is wrong, missing, or not loaded properly. Fix: Check the key is correctly set in your environment variable or .env file — no extra spaces, correct capitalization.
PermissionError: [Errno 13] Permission denied
Meaning: The script doesn't have permission to read/write to that file or folder. Fix: Check file permissions, or run the terminal as Administrator (Windows) / with sudo (Mac/Linux) — but only when you understand why.

AI tools and APIs often use configuration files to store settings. The most common formats you'll encounter:

.env — Environment Variables file
ANTHROPIC_API_KEY=sk-ant-yourkey
MODEL_NAME=claude-sonnet-4-5
DEBUG=false
Stores secrets and configuration outside your code. Never commit .env to version control. Add .env to your .gitignore file.
.json — Configuration / API response
{
"model": "claude-sonnet-4-5",
"max_tokens": 1000,
"temperature": 0.7
}
Structured data format. Used for API responses, model config files, and tool settings. Readable in any text editor.
requirements.txt — Python dependencies
anthropic>=0.25.0
pandas==2.1.0
python-dotenv>=1.0.0
Lists every Python library an AI project needs. Install all at once with pip install -r requirements.txt. Always run this when setting up a new AI project.
