Hardware
vs. Software

Every computer is made of physical components and the programs that run on them. Mastering this distinction is the foundation of all tech literacy.

Core Concept

The Fundamental Divide

Hardware refers to every physical part of a computer — components you can touch, hold, plug in, or replace. If it breaks, you need a technician or a new part. Software refers to the programs, applications, and data that run on that hardware. Software has no physical form — it's instructions written in code.

This distinction matters every day at work: when something goes wrong, knowing whether it's a hardware or software problem helps you describe the issue accurately to IT, attempt the right fix, and set realistic expectations for how long repairs will take.

Analogy
Think of a piano. The piano itself — the keys, strings, frame — is hardware. The music being played, the sheet music, the musical techniques — those are software. You can tune the piano (hardware fix), or learn a new song (software update). One is physical; the other is instruction.

Hardware or software problem?

The situation
Your laptop screen goes black and stays black. The power light is on. You can hear the fan. Pressing keys does nothing.

🔧 Hardware problem. The display itself (the physical screen panel or its connector cable) has likely failed. The computer is running — the OS and software are working — but the output hardware is broken. You need a technician to inspect the screen hardware.

The situation
Microsoft Word keeps crashing every time you try to open a specific document. Other documents open fine. Your computer runs normally otherwise.

💾 Software problem. The application (Word) or the specific file is corrupted — both are software issues. Try repairing the Word installation, or the file itself may be damaged. No hardware involved.

The situation
Your computer is running very slowly. It takes 3 minutes to boot up and apps take forever to open.

Could be either. Software causes: too many startup programs (software fix: disable them), OS needs updates, or malware is running in the background. Hardware causes: failing hard drive (replace), insufficient RAM (upgrade), or overheating CPU (clean vents). Start with software diagnostics before assuming hardware.

The situation
You plug in a USB mouse and nothing happens. The computer doesn't recognize it at all.

Could be either. First test: try the mouse on another computer. If it works there → hardware problem with your USB port. If it doesn't work anywhere → the mouse hardware is broken. If it worked yesterday → possibly a driver issue (software). Always isolate the variable first.

The situation
You get an error: "Not enough disk space to save the file." You have a 1TB hard drive.

Software/data problem. The physical hard drive exists and works (hardware is fine), but the storage space is full — too many files, downloads, or the OS taking up space. Delete unneeded files, empty the Recycle Bin, or move files to cloud storage. No hardware repair needed.

Advanced Concept

The gray area: Firmware

Firmware is software that is permanently (or semi-permanently) installed on hardware chips. It's the lowest level of software — code that tells hardware how to start up before the operating system even loads.

Examples: your computer's BIOS/UEFI (the chip that runs when you first power on), the firmware in your printer that controls its basic functions, or the software inside your keyboard that defines what each key does.

Firmware updates are released by manufacturers to fix bugs or improve performance. Unlike regular software, updating firmware incorrectly can "brick" (permanently damage) a device — so always follow manufacturer instructions carefully.

Why it matters at work
If IT says "we're pushing a firmware update to your laptop," they're updating low-level chip software — not your apps. Your files won't be affected, but you'll need to restart, and the update process takes longer than a normal app update. Don't interrupt it.
AI Readiness — New

🤖 Can your hardware run local AI?

Many modern AI tools (like running a local language model, image generator, or transcription engine) require significant GPU power. Understanding your hardware's capabilities determines whether you can run AI locally or need to use cloud-based APIs instead.

GPU Capability Tiers for Local AI
🟢
CAPABLE — Dedicated GPU with 8GB+ VRAM
Examples: NVIDIA RTX 3070/4060/4070, AMD RX 6700 XT. Can run mid-size language models (7B–13B parameters), local image generation (Stable Diffusion), and real-time transcription. Suitable for serious AI development work.
🟡
LIMITED — Integrated GPU or older dedicated GPU (4GB VRAM)
Examples: Intel Iris Xe, NVIDIA GTX 1050, Apple M1 integrated. Can run small models (1B–3B parameters) slowly, or quantized (compressed) versions of larger models. Good for learning but bottlenecked for production AI tasks.
🔴
CPU-ONLY / NO DEDICATED GPU
Local AI inference on CPU only is extremely slow — a task taking 2 seconds on GPU may take 5–10 minutes on CPU. For learning and API-based work this is fine, but running local models will be impractical. Cloud-based AI APIs (OpenAI, Anthropic, Google) are the right approach here.

RAM Check

Method 1 — Task Manager: Press Ctrl+Shift+Esc → Performance tab → Click "GPU 0" (and GPU 1 if you have two). You'll see the GPU name, current usage, and critically: Dedicated GPU Memory (VRAM). This is the key number for AI tasks.

Method 2 — Device Manager: Press Win+X → Device Manager → Display Adapters. This shows your GPU model name. Search that name online to find its VRAM specs.

VRAM threshold: 8GB+ VRAM = capable for most local models. 4GB = limited. Under 4GB = use cloud APIs.

Apple Silicon (M1/M2/M3/M4): Click Apple menu → About This Mac. Apple Silicon chips use unified memory shared between CPU and GPU — there's no separate VRAM figure. An M1 with 16GB unified memory performs comparably to a dedicated GPU with ~8GB VRAM for many AI tasks. M2/M3/M4 chips are significantly faster.

Intel Mac with AMD GPU: Apple menu → About This Mac → System Report → Graphics/Displays. Look for "VRAM (Dynamic, Max)." Many older Intel MacBooks have only 1.5–2GB which limits local AI tasks to very small models.

NVIDIA GPU: Open Terminal and run nvidia-smi. This shows GPU model, total VRAM, currently used VRAM, and GPU utilization. If nvidia-smi isn't installed: sudo apt install nvidia-utils-xxx (replace xxx with your driver version).

AMD GPU: Run lspci | grep -i vga to see your GPU model, then glxinfo | grep "Video memory" for VRAM. Alternatively sudo apt install radeontop for AMD monitoring.

Any GPU: lshw -C display shows display hardware details including memory size.

AI models also require substantial system RAM (not VRAM). A 7B parameter model typically needs 8–16GB of system RAM minimum. Check your RAM:

Windows: Ctrl+Shift+Esc → Performance → Memory. Look at "Installed memory" and "Speed."

Mac: Apple menu → About This Mac → shows Memory directly.

Linux: Run free -h in Terminal.

Minimum for local AI: 16GB RAM recommended. 8GB is workable for small models. Under 8GB, use cloud-based AI APIs — trying to run local models will cause your system to freeze.

Key Takeaway
If your hardware doesn't meet local AI requirements, that's completely fine — cloud-based AI APIs (OpenAI, Anthropic, Google Gemini) let you access powerful models without local GPU requirements. Understanding your hardware simply tells you which approach to use. Module 4 covers cloud/API basics.
🔒 Security Snapshot

Hardware risks you need to know

Even if you skip the full Cybersecurity module, these hardware-related security concepts are essential for anyone working with AI tools and sensitive data.

⚠️ Unknown USB drives are a malware vector
Never plug in a USB drive you found, received unexpectedly, or can't verify. Malicious USBs can install malware the moment they're connected — before any software runs. This is a real attack technique called "USB drop attack." Only use USB drives from trusted, known sources.
⚠️ Hardware can leak data even when "off"
Hard drives store data even after deletion until overwritten. Disposing of old computers without proper disk wiping (not just reformatting) leaves sensitive data recoverable. Use full-disk encryption (BitLocker on Windows, FileVault on Mac) so that if a device is lost or stolen, the data is unreadable.
⚠️ AI model weights can contain embedded malware
When downloading local AI models (e.g., from Hugging Face), only use models from verified, reputable sources. Malicious actors have published model files containing embedded malicious code. Stick to well-known model providers and check community verification badges before downloading.
