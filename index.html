TechReady Series for Tim · 5 Modules

The foundation for
working with AI tools.

AI tools like large language models don't operate in a vacuum — they work with files, folders, code, operating systems, and cloud services. This series gives students entering a course on managing with AI the grounding they need to interact confidently with those tools: uploading documents, reading file paths, understanding where data lives, and knowing what's happening under the hood when an AI completes a task. Use these modules to fill gaps or to demonstrate baseline tech familiarity before the AI work begins. Cybersecurity is its own extensive standalone module, separate from the condensed overview.

Hardware vs. Software

AI tools run on real machines. Understanding the difference between hardware and software — CPU, RAM, storage, GPU — helps you make sense of why a model runs slowly, why your device struggles with certain tasks, and what IT means when something needs a fix. The GPU in particular is central to how AI inference works.

CPU · RAM · Storage Drivers & Firmware Fault Diagnosis

Operating Systems

AI tools are software, and software runs on an operating system. Knowing whether you're on Windows, macOS, or Linux — and what version — matters when installing AI applications, running local models, troubleshooting compatibility issues, and following setup instructions that differ by platform.

Windows · macOS · Linux Finding Your Version Security Updates & EOL

Module 03 of 05
📁

Files & Folders

Nearly every AI interaction involves a file — uploading a document for analysis, pointing a model at a folder of data, saving outputs, or sharing results. Understanding file paths, extensions like .pdf, .csv, and .txt, and the difference between local and cloud storage is essential for getting things in and out of AI tools reliably.

File Paths & Extensions Local vs Cloud vs Network Naming · Backup · Search

Internet & Cloud Basics

Almost every AI tool is a cloud service — ChatGPT, Claude, Copilot, Gemini, and the rest all live on remote servers you reach over the internet. Understanding how URLs work, what APIs are, the difference between browser-based and locally-run tools, and how cloud sync works makes you a much more effective user and collaborator when AI is in the workflow.

Wi-Fi · DNS · VPN M365 · Teams · OneDrive Troubleshooting

Data Privacy & Cybersecurity

AI tools introduce new attack surfaces — prompt injection, data leakage, oversharing sensitive documents with cloud models, and shadow IT risks when employees use unapproved AI services. This module builds the security awareness needed to use AI responsibly: protecting credentials, understanding what data you should and shouldn't feed a model, and recognizing when something has gone wrong. Standalone — not in the condensed overview.
