The foundation for
working with AI tools.

AI tools don't operate in a vacuum — they work with files, operating systems, hardware, cloud services, and code. This series gives students entering a course on managing with AI the grounding they need: uploading documents, reading file paths, understanding where data lives, running local models, calling APIs, and knowing what's happening under the hood. Now with a diagnostic pre-assessment, AI-readiness expansions in every module, Security Snapshots throughout, and a hands-on capstone scenario.

Module 05 of 05 · Standalone
🔐
Data Privacy & Cybersecurity

AI tools introduce new attack surfaces — prompt injection, data leakage, oversharing sensitive documents with cloud models, and shadow IT risks. This module builds the security awareness needed to use AI responsibly: protecting credentials, understanding what data you should and shouldn't feed a model, and recognising when something has gone wrong. New: an Understanding Code section teaches you to read scripts, use the terminal, decode error messages, and work with config files — foundational skills for every AI interaction.

Phishing · Ransomware Passwords & MFA GDPR · HIPAA · CCPA 🤖 Understanding Code

Who this is for

🤖
AI Course Students

Enrolled in a course on managing with AI and need the tech foundation to actually use the tools — not just talk about them.

📂
Working with Files & Data

AI tools live and die by the inputs you give them. Knowing how to find, format, and share files means you can put the right data in front of a model every time.

☁️
Cloud-Based Workflows

Most AI tools are cloud services. Understanding APIs, sync, and how the internet works helps you integrate AI into real tasks without getting stuck on the plumbing.

🔐
Responsible AI Use

Cybersecurity matters more, not less, when AI is involved. Security Snapshots in every module cover what data is safe to share with a model — and what absolutely isn't.
