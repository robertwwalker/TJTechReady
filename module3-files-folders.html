Core Concept

Files, Folders, and File Systems

Every piece of data on a computer is stored as a file — a named container holding data (text, images, code, video). Files are organized into folders (also called directories), which can contain other folders, creating a tree-like hierarchy. This entire structure is managed by the file system (NTFS on Windows, APFS on Mac).

Understanding this hierarchy means you always know where to find your work and can communicate clearly about file locations — critical when sharing documents with colleagues or troubleshooting with IT.

Analogy
Think of a filing cabinet. The cabinet itself is your hard drive. Drawers are top-level folders. Within each drawer are hanging folders (sub-folders). Inside those are individual documents (files). A file path is like the address: "Cabinet A → Drawer: Finance → Folder: Q3 2024 → Document: Budget.xlsx"

File Extensions

What does the file extension tell you?

The letters after the final dot in a filename (.docx, .pdf, .jpg) are the extension. They tell the OS what type of data the file contains and which app should open it. Knowing extensions prevents confusion and helps you share files in the right format.

Where Files Live

Local, Network, and Cloud Storage

💻
Local Storage (Your Computer)
Files saved directly to your hard drive or SSD. Fast access, works without internet. Located in C:\Users\YourName\ (Windows) or /Users/yourname/ (Mac). Risk: if your computer fails or is stolen without backup, files can be lost.

Practical Skills

Finding lost files & good practices

Finding lost files
Naming conventions
Version control
Backup strategy

Windows: Use the search box in File Explorer (top right). You can also press Win+S to search the entire computer. Filter by date modified, file type, or size. Check the Recent items list in most apps (File → Recent).

Mac: Use Spotlight (Cmd+Space) and type the filename. Or in Finder, press Cmd+F to search within a specific folder. Check Recents in the Finder sidebar.

If you accidentally deleted a file: Check the Recycle Bin (Windows) or Trash (Mac) before panicking — files go there first. Right-click → Restore.

Good file naming prevents confusion, especially in shared drives. Use dates in filenames for version tracking: 2024-03-15_Budget_Final.xlsx (YYYY-MM-DD sorts chronologically). Avoid spaces — use underscores or hyphens. Avoid special characters (/ \ : * ? " < > |) as they cause issues across systems. Be specific: Smith_Contract_Signed.pdf not Document1.pdf.

When multiple versions of a file exist, naming matters: Report_v1.docx, Report_v2_FINAL.docx, Report_v2_FINAL_JSmithEdits.docx. This gets unwieldy. Better: use cloud tools like Google Docs or SharePoint that have built-in version history — you can see every change ever made and restore any previous version. Check: File → Version History in Google Docs or Microsoft Office 365.

The 3-2-1 backup rule: keep 3 copies of important data, on 2 different types of storage, with 1 copy offsite (or in cloud). For most workers: your files on your computer (1), synced to cloud storage like OneDrive (2, offsite), and optionally on a USB drive or network drive (3). Most companies handle backup automatically — but verify with IT what's actually backed up vs. what's only on your local drive.

Scenario
You saved a report yesterday but can't find it today. You search your Desktop and Documents — nothing.

Systematic approach: (1) Check the app's Recent Files (File → Recent). (2) Search by filename in File Explorer or Spotlight. (3) Search by date — you know you saved it yesterday. (4) Check if you were working in a cloud folder (OneDrive, Google Drive) — it may have synced to the cloud. (5) Check Downloads folder — sometimes files default there.

Scenario
A client asks you to send them a document as a PDF, but you only have the .docx Word file.

Convert to PDF: In Microsoft Word, go to File → Save As (or Export) → choose PDF as the format. Or File → Print → choose "Microsoft Print to PDF" as the printer. On Mac: File → Print → PDF → Save as PDF. PDFs preserve formatting across all devices and can't be accidentally edited by the recipient — that's why they're preferred for final documents.

Scenario
Your manager says "put the final version in the shared drive, in the Projects folder." You're not sure where the shared drive is.

On Windows, look in File Explorer's left sidebar for a mapped drive letter (Z:, S:, etc.) or a network location. On Mac, check Finder's sidebar under "Network." If you can't find it, ask your manager for the exact path (e.g., \\server\Projects) or if it's in a cloud service like SharePoint, they'll send you a link. Never assume you've found the right location — confirm before replacing important files.

Scenario
You need to email someone a folder containing 30 files. Your email rejects the attachment because it's too large.

Options: (1) Compress the folder — right-click the folder → "Compress" (Mac) or "Send to → Compressed zip folder" (Windows). This creates a single .zip file, often 30-60% smaller. (2) Share via cloud link — upload to Google Drive, OneDrive, or Dropbox, then share the link. This is better for large files and keeps them editable. (3) Use a service like WeTransfer for very large files.
